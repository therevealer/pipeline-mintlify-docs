</Info>

PipelineCI is a CI/CD automation API that lets you programmatically trigger builds, monitor jobs, manage deployment environments, and receive real-time webhook notifications - all without touching a UI.

Whether you're building an internal developer portal, automating deployments from a script, or integrating CI/CD into your own platform, PipelineCI gives you full control over your software delivery pipeline.

## Before you start

<Steps>
  <Step title="Get your API key">
    Sign in to your [PipelineCI dashboard](https://dashboard.pipelineci.dev) and navigate to **Settings → API Keys** to generate your key.
  </Step>
  <Step title="Read the authentication guide">
    Learn how to include your API key in requests. See [Authentication](/concepts/authentication).
  </Step>
  <Step title="Make your first request">
    Try listing your pipelines to confirm everything is working:

    ```bash
    curl https://api.pipelineci.dev/v1/pipelines \
      -H "X-API-Key: your_api_key_here"
    ```
  </Step>
</Steps>

## Base URL

All API requests are made to:

```text
https://api.pipelineci.dev/v1
```

A sandbox environment is also available for testing:

```text
https://sandbox.api.pipelineci.dev/v1
```

<Note>
  Requests made to the sandbox environment do not trigger real deployments and will not affect your production infrastructure.
</Note>
